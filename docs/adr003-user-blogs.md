# Блоги пользователей

Проблема
==
У пользователей JoinRpg есть блоги (Telegram-каналы и т.п.), связанные с ролевыми играми. Сейчас нет способа найти эти блоги через JoinRpg. Хочется дать пользователям возможность указать свои блоги и предоставить публичный каталог блогов.

Решение
==
Пользователь добавляет ссылки на свои блоги в профиле. Система сама определяет тип блога и подтягивает метаданные (название, описание, число подписчиков) из внешних API.

Модель данных
---
Новая сущность `UserBlog`:
- `UserBlogId` (PK)
- `UserId` (FK)
- `BlogUri` (уникальный ключ)
- `BlogType` (enum: Telegram, и т.п.)
- `BlogName`, `BlogDescription`, `SubscriberCount` — заполняются из внешнего API
- `IsVerified` — подтверждён ли блог

Редактирование
---
Пользователь добавляет/удаляет блоги через `ManageController.SetupProfile`. Бизнес-логика вынесена в отдельный `UserBlogService`.

Новый проект JoinRpg.Web.UserProfile
---
Blazor-компоненты для отображения блогов (по образцу JoinRpg.Web.GameSubscribe):
- `UserBlogCard.razor` — карточка одного блога
- Список блогов пользователя

Доступ к данным по образцу существующих клиентов:
- `IUserInfoClient` — интерфейс (реализуется на клиенте через API и на сервере напрямую)
- `UserInfoViewService` — серверная реализация
- `WebApi.UserInfoController` — API-контроллер

(по аналогии с `IPlotClient` / `PlotViewService`)

Контроль доступа
---
- `UserProfileAccessReason` — причина доступа к данным профиля
- Блоги доступны публично: `ContactsAccessType.Public`

Публичная страница /blogs
---
Страница со списком всех блогов. Используется `JoinQuickGrid` (обёртка над QuickGrid) с пагинацией по 20 записей.

Интеграция с Telegram
---
Для Telegram-блогов данные (название, описание, число подписчиков) подтягиваются через Telegram Bot API.

Обновление запускается еженощной джобой (см. [adr002-dailyjobs.md](adr002-dailyjobs.md)).
