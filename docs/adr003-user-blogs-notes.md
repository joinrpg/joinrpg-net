# Блоги пользователей — заметки по реализации

Детали, не вошедшие в основной ADR.

## Размещение сущностей
- Enum `BlogType` — в `PrimitiveTypes`
- Сущность `UserBlog` — в `JoinRpg.DataModel`

## URL: валидация и нормализация
- При добавлении блога пользователь вводит URL
- URL нормализуется (убираются лишние слеши, параметры и т.п.)
- Upsert-логика: если блог с таким `BlogUri` уже есть у пользователя — обновляем, иначе создаём

## Автозаполнение полей
- `BlogName` заполняется из `BlogUri` (например, `@channel_name` для Telegram)
- `BlogType` определяется автоматически по анализу URL (telegram.me → Telegram, и т.п.)
- `BlogDescription` и `SubscriberCount` изначально пустые, заполняются джобой

## Тесты
- Юнит-тесты на валидацию/нормализацию URL и определение `BlogType`
- Интеграционный тест в `JoinRpg.IntegrationTest` для `ManageController.SetupProfile` (добавление блога через профиль)

## Колонки таблицы на /blogs
- Название блога
- Описание
- Число подписчиков

## Telegram Bot API
- Библиотека: https://telegrambots.github.io/
- Используется для получения метаданных канала (название, описание, число подписчиков)

## Организация по проектам
- Все Blazor-компоненты блогов живут в `JoinRpg.Web.UserProfile`
